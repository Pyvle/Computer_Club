# Ручные сценарии тестирования мобильного приложения

Эти сценарии относятся к интеграционному и UI-тестированию мобильной части системы
«Приложение управления компьютерным клубом». Они не запускаются как JUnit-тесты,
потому что требуют Android-эмулятор, backend, Redis/PostgreSQL и внешние SDK.

| № | Сценарий | Ожидаемый результат |
|---|---|---|
| M-01 | Открыть приложение и дождаться загрузки списка клубов | Отображаются активные клубы, карточки содержат название, адрес и стоимость |
| M-02 | Переключиться на карту клубов | Клубы отображаются на карте по координатам, выбранный клуб открывается из маркера |
| M-03 | Открыть карточку клуба | Отображаются описание, адрес, характеристики мест, тарифы и товары клуба |
| M-04 | Начать бронирование без авторизации | Пользователь перенаправляется на OTP-вход и после входа возвращается к бронированию |
| M-05 | Выбрать дату, время и пакет времени | Конец бронирования пересчитывается по выбранному пакету |
| M-06 | Выбрать место на схеме зала | Свободное место выбирается, занятое или конфликтующее место недоступно |
| M-07 | Добавить бронирование в корзину | В корзине появляется строка бронирования с местом, временем и суммой |
| M-08 | Добавить товар из магазина в корзину | Количество товара и сумма корзины пересчитываются |
| M-09 | Оформить корзину | Создается покупка, корзина очищается, запись появляется в истории |
| M-10 | Открыть историю и профиль | Показываются бронирования, покупки и суммы по текущему пользователю |
| M-11 | Отправить жалобу на клуб | Сообщение отправляется и становится доступным в административной части |
| M-12 | Выйти из профиля | Токены очищаются, закрытые разделы снова требуют авторизацию |
